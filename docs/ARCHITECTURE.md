# 🏗️ Architecture — eSports Analytics Dashboard

## Overview

```
┌─────────────┐     ┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│  MySQL 8.0  │────▶│   EXTRACT    │────▶│   VALIDATE   │────▶│  TRANSFORM   │
│  (9 tables) │     │  (9 queries) │     │  (9 schemas) │     │  (dict → JSON)│
└─────────────┘     └──────────────┘     └──────────────┘     └──────┬───────┘
                                                                      │
                    ┌──────────────┐     ┌──────────────┐             │
                    │  GitHub      │◀────│    LOAD      │◀────────────┘
                    │  Pages       │     │  (JSON file) │
                    └──────────────┘     └──────────────┘
```

## Database Schema

| Table | Description | Key Columns |
|---|---|---|
| `paises` | Países representados | id, nombre |
| `equipos` | Equipos de eSports | id, nombre, pais_id |
| `jugadores` | Jugadores individuales | id, nombre, nacionalidad, fecha_nacimiento |
| `jugador_equipos` | Relación jugador-equipo | jugador_id, equipo_id, rol |
| `competencias` | Torneos y competencias | id, nombre, tipo, premio_total |
| `competencia_equipos` | Equipos en competencias | competencia_id, equipo_id, posicion |
| `rosters` | Alineaciones por competencia | jugador_id, equipo_id, competencia_id |
| `partidos` | Partidos jugados | id, competencia_id, fecha |
| `partido_equipos` | Equipos en partidos | partido_id, equipo_id, resultado |
| `estadisticas_jugador` | Performance individual | jugador_id, partido_id, kills, deaths, assists |

## ETL Pipeline (`src/etl/pipeline.py`)

### Extract Phase
Executes 9 SQL queries against MySQL, each producing a Pandas DataFrame:
1. **kpis** — Métricas generales (total equipos, jugadores, premios)
2. **ranking_paises** — Ranking por premios acumulados
3. **top_jugadores** — Top 5 jugadores por winrate 2024
4. **evolucion** — Comparativa 2024 vs 2025
5. **roles** — Análisis Titular vs Suplente
6. **top_equipos** — Top 5 equipos por competencias
7. **competencias** — Detalle de torneos
8. **veteranos** — Jugador más veterano por equipo
9. **metricas** — Métricas resumen globales

### Validate Phase (`src/etl/validators.py`)
Each dataset passes through business rule checks:
- Numeric ranges (winrates 0-100%, ages 15-50)
- Non-negative values (prizes, counts)
- Allowed values (roles, competition types)
- Non-null required fields

### Transform Phase
Converts raw DataFrames into the JSON structure consumed by the frontend.

### Load Phase
Writes `datos-dashboard.json` to `src/frontend/assets/data/`.

## CI/CD (`deploy.yml`)

```yaml
Trigger: push to main
Jobs:
  1. etl:
     - MySQL 8.0 service container
     - Load schema.sql
     - Python 3.12 + pip cache
     - Run pipeline.py
     - Upload frontend artifact
  2. deploy:
     - Deploy to GitHub Pages
```

## Frontend (`src/frontend/`)
Single-page dashboard using Chart.js for data visualization. Consumes `datos-dashboard.json` generated by the ETL pipeline.
